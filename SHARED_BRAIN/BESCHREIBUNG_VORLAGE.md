# BESCHREIBUNG VORLAGE

## So schreibst du gute Beschreibungen

### Struktur:
```
1. KURZE DEFINITION (1 Satz)
   - Was es ist, in einem Satz

2. ERKLÄRUNG (2-3 Sätze)
   - Warum es wichtig ist
   - Was es bewirkt

3. BEISPIEL (optional)
   - Praktisches Beispiel aus der Realität
```

---

## Beispiel: Project Charter

### 1. Definition:
> Ein Project Charter ist das Fundament deines Projekts.

### 2. Erklärung:
> Ohne dieses Dokument startest du oft im Blindflug. Es stellt sicher, dass alle Stakeholder dasselbe Ziel haben, legt Befugnisse fest, und schützt vor Scope Creep.

### 3. Nutzen:
> Formale Zusage für Budget, Personal und Zeit - schwarz auf weiss.

## Checkliste für gute Beschreibungen:

- [ ] Ein Satz der das Wesentliche trifft
- [ ] Erklärung warum wichtig
- [ ] Praktischer Nutzen klar
- [ ] Kurz und prägnant (nicht zu lang)
- [ ] Verständlich für Alle
