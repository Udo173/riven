# Prompt Engineering - Skill Guide

## Was ist Prompt Engineering?

Die Kunst, Anweisungen zu schreiben die ein AI Modell effektiv befolgen kann.

> "Der Unterschied zwischen einem vagen und präzisen Prompt ist keine Klugheit. Es ist Struktur."

---

## Grundlegende Techniken

### 1. Rolle Definieren
```
Du bist ein [Rolle] mit [Erfahrung].
```

### 2. Kontext Geben
```
Situation: [Hintergrund]
Ziel: [Was erreicht werden soll]
```

### 3. Klare Anweisungen
```
TUE:
- [Schritt 1]
- [Schritt 2]

TUE NICHT:
- [Vermeiden]
```

### 4. Format Angeben
```
Antworte als:
- JSON
- Markdown
- Liste
- Tabelle
```

### 5. Beispiele Geben (Few-Shot)
```
Beispiel:
Eingabe: X
Ausgabe: Y

Jetzt:
Eingabe: [Dein Input]
```

---

## Fortgeschrittene Techniken

### Chain-of-Thought (CoT)
```
Denke Schritt für Schritt durch das Problem.
```

### Zero-Shot CoT
```
Lass das Modell erklären wie es zur Antwort kommt.
```

### Tree of Thoughts
```
Erkunde mehrere Lösungswege bevor du dich festlegst.
```

### Self-Consistency
```
Lasse das Modell die Aufgabe mehrfach lösen 
und wähle die konsistenteste Antwort.
```

---

## Best Practices

### DO ✅
- Spezifisch und präzise sein
- Kontext und Hintergrund geben
- Struktur vorgeben
- Beispiele nutzen
- Erwartetes Format definieren
- Einschränkungen klar machen

### DON'T ❌
- Vage oder mehrdeutig
- Zu viele Informationen auf einmal
- Widersprüchliche Anweisungen
- Annahmen nicht überprüfen

---

## Prompt Vorlagen

### Für Code
```
DU BIST: Erfahrener [Sprache] Entwickler

AUFGABE: [Beschreibe was der Code tun soll]

ANFORDERUNGEN:
- [Anforderung 1]
- [Anforderung 2]

FORMAT: [Code mit Kommentaren]
```

### Für Analyse
```
DU BIST: Analytiker mit Erfahrung in [Bereich]

KONTEXT: [Hintergrundinformation]

FRAGE: [Was analysiert werden soll]

FORMAT: 
- Hauptpunkte
- Detaillierte Erklärung
- Empfehlungen
```

### Für Kreativ
```
KONZEPT: [Idee/Beschreibung]

STIL: [Gewünschter Stil]

TON: [Tonfall]

FORMAT: [Gewünschtes Format]
```
