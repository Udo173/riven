# MONITOR & CONTROL - PROJECT MANAGEMENT

---

## ORIGINAL TEXT:

**Key concepts**

**1.** During the monitor and control the project phase, project managers ensure that the team executes the project according to the project objectives.

**2.** Project monitoring is the process of continuously tracking, recording, and evaluating the progress of project activities so that PMs can keep the clients and internal stakeholders informed of the project's status.

**3.** Project controlling is the process of hiring necessary actions to address issues or changes identified through monitoring. PMs repeat the project controlling process many times throughout the project lifecycle by following four steps: Step 1: Establish standards, Step 2: Observe performance, Step 3: Compare planned with actual performance, Step 4: Take corrective action.

**4.** PMs manage the project's team schedule according to the baseline plan and if required, adjust the baseline plan. To manage the team's schedule, PMs gather progress updates, track and compare progress, identify and mitigate risks, and report progress.

**5.** Scope creep refers to the gradual expansion of project scope beyond its original objectives, requirements, or deliverables, without corresponding adjustments to time, budget, or resources. To manage the project team's tasks and avoid scope creep, PMs update the project's status and monitor its progress, identify deviations from plan, and implement change control.

**6.** The project makes more profit if PMs work within the budget. To manage the project costs, PMs follow several strategies, such as creating a budget plan, monitoring project expenses and costs, and optimizing resource allocation.

**7.** Change management is a formal process that helps PMs record, track, and resolve changes that occur during the project execution phase, ensuring they are accepted, deferred, or rejected. PMs effectively manage changes by following four steps: Step 1: Identify change, Step 2: Investigate the change, Step 3: Estimate costs, Step 4: Approve the change.

**8.** A project status report is a formal document that provides updates on the progress of a project, compares it to the project plan, lists planned actions, and highlights issues or risks. PMs create a project status report that includes various sections to keep the stakeholders and team members informed.
